# Система предсказания популярности поста

__Disclaimer__: проект создан исключительно в образовательных целях

## Этапы работы над проектом
Каждый из этапов хранится в отедльном файле, чтобы можно было легко отследить результаты каждого из них.

Выделим основные:

1.__Сбор данных из телеграм-каналов__ (постов с информацией о количестве просмотров, комментариев, реакций):

   были выбранны интересующие меня тематики: новости, книги. В общей сложности был собран датафрейм из 5800 строк. Он состоял из столбцов с датой публикации, текстом поста, количеством просмотров, реакций, комментариев. Возможно, необходимо было выбрать каналы с большей популярностью и активностью подписчиков, чтобы результат был лучше;

2. __Анализ собранных данных__ (построение графиков для понимания общих тенденций):

    на основании полученных графиков были сделаны выводы о времени пиковой активности пользователей в каналах, соотношении лайков и комментариев;

3. __Обработка данных для более точной работы моделей__:

    была проведена очистка текста с помощью регулярных выражений, приведение к нижнему регистру, токенизация, лемматизация и удаление стоп-слов. Было принято решение оставить эмодзи в постах, чтобы учитывать и их тоже;

4. __Векторизация предобработанных данных__:
   
   была проведена векторизация с помощью 3х методов (пробы ради). В конечном итоге был выбран вариант word2vec;
   
5. __Применение моделей машинного обучения для классификации__:

   были применены стандартные модели машинного обучения для классификации: логистическая регрессия, метод опорных векторов и метод ближайших соседей. В качестве целевой переменной было выбрано следующее соотношение: (кол-во комментариев + кол-во реакций)/общее кол-во просмотров.
   наибольшую точность показали первые 2 алгоритма (с точностью ~0.69), что в целом не плохо, однако для достижения лучших метрик качества стоит заняться подбором гиперпараметров.
